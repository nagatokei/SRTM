SRTM ユーザーガイド / README

このページでは、SRTM（Webカメラフェイストラッキングシステム）の詳細な設定やトラブルシューティングについて解説します。

⚙️ フェイストラッキングについて

本システムはVRM 1.0に対応しており、以下の仕様で動作します。

  • トラッキング項目: 目、鼻、口、頬の位置および頭部の回転情報。
  • パーフェクトシンク: 対応モデルでは最大48か所のポイントを使用し、豊かな表情を再現します。
  • 軽量簡易モード: 動作にラグがある場合は「拡張表情モード」のチェックを外してください。
  • カメラ権限: 起動時にブラウザのカメラ使用を許可してください。
  • スリープ防止: 送信機側をアクティブな小さなウィンドウで表示させておくと、OSによるスリープを防げます。

🚀 動作環境とパフォーマンス

「映像を送らず、解析データのみを送る」独自設計により、圧倒的な軽量動作を実現しています。

グレード 想定デバイス 通信環境

Standard 一般的なPC / iPhone 11以降 / SnapDragon 7系 Wi-Fi / 4G / 5G

💡 技術的なアドバンテージ
通常のビデオ通話が毎秒 2,000KB 以上のデータを送るのに対し、本システムはわずか 1KB 前後。 送信側（Sender）はカメラ映像を描画しない「ステルス解析モード」を採用し、計算リソースを解析に集中させています。

❓ よくある質問 (FAQ)

Q.ローカルファイルアクセスの壁があるはずだ (File System API)。

A. File API / IndexedDBという技術を用いていますので不正アクセスではありません。

Q.クロスドメイン通信の壁を知らないのか (CORS / Mixed Content)。

A. WebRTCはブラウザのセキュリティモデルに基づいて設計されており、クロスドメイン通信を安全に処理します。

Q.リソース占有の壁がある、ブラウザが暴走と判断するはずだ (Hardware Acceleration)。

A. MediaPipe: 外部の怪しいプログラムではなく、Googleが公開している信頼されたライブラリをブラウザ内で動かしています。

🛡️ セキュリティ・ポリシー

本システムは安全性を最優先に設計されています：

  • URLサニタイズ: 外部参照時は厳格なドメイン・拡張子チェックを行い、悪意のあるパスを排除します。
  • サンドボックス動作: すべての処理はブラウザの隔離環境内で完結し、ローカルファイルへの不正アクセスは行いません。
  • データ最小化: P2P通信ではアバター制御に必要な数値データのみを扱い、実行形式のファイルの送受信は行いません。
